# 双设备聊天测试 - 手动测试指南

## 测试场景
测试两个 IM Boy 应用实例之间的消息收发功能。

## 准备工作

### 1. 检查设备
```bash
cd /Users/leeyi/project/imboy.pub/imboyapp
flutter devices
```

预期输出应包含：
- macOS (desktop)
- iPhone 或 iOS 模拟器

### 2. 启动 iOS 模拟器（如果需要）
```bash
flutter emulators --launch apple_ios_simulator
```

## 测试步骤

### 第一步：启动两个应用实例

**Terminal 1 - macOS 应用**
```bash
cd /Users/leeyi/project/imboy.pub/imboyapp
flutter run -d macos
```

**Terminal 2 - iOS 应用**
```bash
cd /Users/leeyi/project/imboy.pub/imboyapp
flutter run -d 00008140-000E30561E32801C  # 使用实际设备 ID
```

或者使用模拟器：
```bash
flutter run -d ios
```

### 第二步：登录账号

#### macOS 实例（账号 1）
1. 等待应用启动完成
2. 点击"登录"按钮
3. 输入手机号：`108@imboy.pub`
4. 输入密码：`admin888`
5. 点击"登录"

#### iOS 实例（账号 2）
1. 等待应用启动完成
2. 点击"登录"按钮
3. 输入手机号：`118@imboy.pub`
4. 输入密码：`admin888`
5. 点击"登录"

### 第三步：添加好友（如果需要）

如果两个账号还不是好友，需要先添加好友：

1. 在 macOS 实例上：
   - 点击"添加好友"
   - 输入对方账号：`118@imboy.pub`
   - 发送好友请求

2. 在 iOS 实例上：
   - 接受好友请求

### 第四步：发送消息测试

#### 测试 1：macOS 发送消息给 iOS

**在 macOS 实例上操作：**
1. 点击"联系人"标签
2. 查找并点击用户 "118"
3. 进入聊天界面
4. 在输入框输入：`Hello from macOS!`
5. 点击"发送"按钮

**验证点：**
- macOS 上显示消息已发送（消息气泡在右侧）
- iOS 上收到新消息通知
- iOS 聊天界面显示 "Hello from macOS!"（消息气泡在左侧）

#### 测试 2：iOS 回复消息

**在 iOS 实例上操作：**
1. 打开与 108 的聊天
2. 在输入框输入：`Hello from iOS!`
3. 点击"发送"按钮

**验证点：**
- iOS 上显示消息已发送
- macOS 上收到新消息通知
- macOS 聊天界面显示 "Hello from iOS!"

#### 测试 3：连续消息测试

**在 macOS 实例上发送多条消息：**
1. 发送：`测试消息 1`
2. 发送：`测试消息 2`
3. 发送：`测试消息 3`

**验证点：**
- iOS 按顺序接收所有消息
- 消息顺序正确
- 消息内容完整

**在 iOS 实例上回复：**
1. 发送：`收到 1`
2. 发送：`收到 2`
3. 发送：`收到 3`

**验证点：**
- macOS 按顺序接收所有回复
- 消息顺序正确

#### 测试 4：特殊字符和表情测试

**测试内容：**
- 发送中文字符：`你好世界`
- 发送表情符号：`😀😁😂`
- 发送特殊符号：`@#$%^&*()`

**验证点：**
- 所有字符正确显示
- 没有乱码或编码问题

## 常见问题

### Q1: 应用启动失败
**解决方案：**
```bash
# 清理并重新构建
flutter clean
flutter pub get
flutter run -d macos
```

### Q2: 消息发送失败
**检查项：**
- 网络连接是否正常
- 后端服务是否运行
- WebSocket 连接是否建立
- 账号是否已添加好友

### Q3: 消息接收延迟
**检查项：**
- 网络延迟
- 服务器负载
- 应用后台运行状态

### Q4: 乱码问题
**检查项：**
- 字符编码设置
- 数据库字符集
- 应用本地化配置
