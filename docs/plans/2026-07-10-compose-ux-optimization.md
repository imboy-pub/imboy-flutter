# 发消息交互简化执行计划（群 / 频道 / 朋友圈）

> 创建：2026-07-10 | 状态：P0/P1 已落地（提交 b60f44c5），P2 后置 | 范围：imboyapp 三处 compose 流程
> 目标：降低"发消息/发布"的步骤数与入口深度，复用已有能力，消除各处重复实现。

---

## 0. 背景与目标

用户反馈：群功能、频道、朋友圈的**发消息体验不够简洁**。经代码实证，根因不是视觉，而是：

1. **入口断层** —— 群协作工具（投票/日程/任务）在聊天内无法直达，埋在群详情页。
2. **步骤割裂** —— 朋友圈媒体入口拆成 4 项、可见性要跳两层。
3. **能力退化 + 串行慢** —— 频道发布无草稿、多媒体串行上传。
4. **该复用未复用（DRY 缺失）** —— `wechat_assets_picker` 多选、`StorageService` 草稿、`chat_input` 表情/@ 各处各写各的。

**总原则**：减少步骤 / 统一入口 / 复用现成能力 优先于视觉重绘。

**全局约束（验收前置）**：
- `dart analyze lib` 必须保持 **No issues found!**（现基线为 0，见 imboyapp/CLAUDE.md）。
- 所有新文案走 `assets/i18n/zh-CN/<namespace>.i18n.yaml` → `dart run slang`，禁止硬编码中文。
- 颜色/间距/字号必须走 `AppColors` / `AppSpacing` / `FontSizeType` Token。
- **功能验收必须真机**（禁模拟器），设备 iPhone 16e `00008140-000E30561E32801C`。

## 3. 执行计划（含落地步骤与验收标准）

### 🔴 P0-1｜群 `+` 面板注入群协作工具

- **问题**：发起群投票需 5 步（聊天→右上角→群详情→功能区→投票）。
- **落地步骤**：
  1. `ExtraItems` 已有 `type`（C2C/C2G/C2S）与 `options`（含 `to`=groupId），无需改签名。
  2. 在 `extra_item.dart` 的 `allItems` 增加 `if (widget.type == 'C2G')` 分支，追加：群投票 / 群日程 / 群任务（可选：群公告）。
  3. 每项 `onPressed` 直接复用现成路由：`context.push('/group/${options['to']}/vote' | '/schedule' | '/task')`。
  4. 新增图标走 `IconTheme`（保持工具属性），文案走 i18n（`group` namespace 已有 vote/schedule/task 键则复用）。
- **涉及文件**：`lib/page/chat/widget/extra_item.dart`（+ 必要时 i18n）。
- **验收标准**：
  - [ ] 群聊中点 `+`，面板出现"群投票/群日程/群任务"图标。
  - [ ] 点击各项能正确跳转到对应群功能页，且携带正确 groupId。
  - [ ] C2C/C2S 会话**不出现**群工具项（回归不串场）。
  - [ ] 从"打开群聊"到"进入发起投票页" ≤ 2 步。
  - [ ] `dart analyze lib` 零告警；真机验证跳转正常。
- **成本**：S（路由现成，纯接线）

### 🔴 P0-2｜朋友圈媒体改统一多选网格

- **问题**：4 项 ActionSheet 割裂，9 图点 9 次单传。
- **落地步骤**：
  1. `_showMediaPicker`（`moment_create_page.dart:337`）改为调用 `AssetPicker.pickAssets`（`wechat_assets_picker`，`requestType: RequestType.common` 支持图/视频混选，`maxAssets` = `momentMaxImageCount`）。
  2. 保留"拍照/拍视频"作为选择器内相机入口（picker 原生支持）或单独一个相机项。
  3. 选中后批量并行上传：`Future.wait(assets.map(_uploadOne))`，复用现有 `_uploadFile`（scope='moment'）。
  4. 视频缩略图/时长逻辑（`:220-233`）抽成 `_uploadOne` 内分支。
  5. 上传中沿用 `_isUploading` + 占位格（`_MediaUploadingPlaceholder`）。
- **涉及文件**：`lib/page/moment/moment_create_page.dart`。
- **验收标准**：
  - [ ] 一次可多选图片/视频（混选），一次批量上传。
  - [ ] 9 张图从选择到全部上传完成，用户操作 ≤ 2 次点击。
  - [ ] 混选校验仍生效（`validateMediaSelection`：图/视频不可混、数量上限）。
  - [ ] 上传失败有单项占位失败反馈，不阻塞其余成功项。
  - [ ] 草稿恢复、可见性、字数统计等既有功能不回退。
  - [ ] 真机验证：相册多选、拍照、拍视频三路径均可发布成功。
- **成本**：M

### 🔴 P0-3｜频道多媒体并行上传 + 整体进度

- **问题**：`for-await` 串行，9 文件排队慢；失败只弹单条 SnackBar。
- **落地步骤**：
  1. `_pickAndSendMedia`（`channel_publish_bar.dart:153`）的 `for` 循环上传改 `Future.wait`，并发上传（保留逐项 payload 组装）。
  2. 上传阶段展示"N/总数 已上传"进度（可复用 `AppLoading` 或输入栏内联进度）。
  3. 全部完成后按序 `publishMessage`，失败项汇总一条提示（含成功/失败计数）。
  4. 并发度可加轻量上限（如 3），避免大文件同时打满带宽（`ponytail:` 固定并发 3，需要再调）。
- **涉及文件**：`lib/page/channel/widgets/channel_publish_bar.dart`。
- **验收标准**：
  - [ ] 9 个文件并行上传，总耗时显著低于串行（真机对比可感知）。
  - [ ] 上传过程有整体进度反馈。
  - [ ] 部分失败时，成功项正常发布，失败项汇总提示且可重试。
  - [ ] `mounted` 检查完整，返回页不崩溃（保留现有保护）。
- **成本**：S

### 🟡 P1-1｜朋友圈可见性单页化 + 记忆

- **问题**：可见性选择跳两层。
- **落地步骤**：
  1. 将 `_showVisibilityPicker`（ActionSheet 五选一）与允许/排除名单合并为一页：顶部选可见性、选"部分可见/不给谁看"时同页展开好友多选。
  2. 用 `StorageService` 记忆上次可见性与名单，下次默认带出。
- **涉及文件**：`moment_create_page.dart:380-409`、`moment_friend_picker/`。
- **验收标准**：
  - [ ] 设置"谁可以看"在**同一页**完成，无二次跳转。
  - [ ] 再次进入发布页，默认带出上次可见性设置。
  - [ ] 五种可见性语义与后端字段映射不变（回归）。
- **成本**：M

- **群工具跳转依赖现有路由稳定性** —— 改动前确认 `/group/:gid/vote|schedule|task` 路由与守卫正常。
